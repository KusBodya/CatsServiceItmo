# Лабораторная работа 5 - Микросервисная архитектура

##  Техническое задание

Микросервисное приложение для управления питомцами и владельцами с асинхронной коммуникацией через Apache Kafka.

##  Архитектура

Система состоит из **трех независимых микросервисов**:

### 1. Pets Service
- Управление питомцами
- Хранит только `ownerId` (внешний идентификатор)
- БД: PostgreSQL (отдельная)

### 2. Owners Service  
- Управление владельцами
- Не содержит данных о питомцах
- БД: PostgreSQL (отдельная)

### 3. HTTP Web Gateway
- REST API для клиентов
- Аутентификация и авторизация
- Агрегация данных от микросервисов
- БД: PostgreSQL (пользователи, роли)

##  Стек технологий

- **Java** 17+
- **Spring Boot** 3.x
- **Maven** - сборка проекта
- **Apache Kafka** - брокер сообщений
- **Spring Kafka** - KafkaTemplate
- **Spring Web** - REST API
- **Spring Data JPA** - работа с БД
- **PostgreSQL** - база данных
